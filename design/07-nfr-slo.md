# NFRs / SLOs والحدود الرقمية

> ✅ **قيم معتمدة (`OQ‑02` · توقيع 2026‑07‑10)**: كل الأرقام أدناه **معتمدة ضمن `OQ‑02`** لمنصّة تخدم «آلاف المستخدمين». تُنشر عبر `LIMITS_*`/`SLO_*` في الإعداد المركزي (لا مُثبَّتة في الكود) وتُعاير بالقياس — تبقى قابلة للتعيير.

## 1) أهداف الجاهزية (Availability)
| المكوّن | الهدف | ملاحظة |
|---------|-------|--------|
| API (المسارات المتزامنة) | 99.9% شهرياً | خلف Nginx، نسخ متعددة بلا حالة |
| البثّ (WS/SSE) | 99.5% | اتصالات طويلة، أكثر حساسية |
| العمّال (مهام ثقيلة) | 99.5% إنجاز | at‑least‑once + DLQ ⇒ لا فقد |

## 2) أهداف زمن الاستجابة (Latency SLO)
| العملية | p50 | p95 | p99 |
|---------|-----|-----|-----|
| قراءة متزامنة (GET مورد) | 40ms | 150ms | 300ms |
| كتابة متزامنة (POST/PATCH) | 80ms | 250ms | 500ms |
| أول رمز في البثّ (agent invoke) | 400ms | 1.2s | 2.5s |
| استرجاع RAG (embed + بحث Qdrant) | 120ms | 400ms | 800ms |
| قبول مهمة ثقيلة (media/index enqueue) | 60ms | 200ms | 400ms |
| فرض/التقاط `usage` (منفذ وارد متزامن) | 5ms | 20ms | 50ms |
| زمن دورة Outbox (نشر بعد الالتزام) | 0.5s | 1.5s | 3s |

> استثناء: زمن نداء مزوّد LLM/الصور خارج SLO التطبيق (تبعية خارجية) — يُقاس ويُعرض منفصلاً.

## 5) الأداء والصيانة (بنيوياً)
- **Statelessness:** لا حالة في RAM للطلب/الوكيل ⇒ توسّع أفقي حرّ (قاعدة الوكلاء).
- **تجميع الاتصالات:** كل الوصول لقاعدة البيانات عبر PgBouncer (Transaction pooling, D‑21) ⇒ سقف اتصالات Postgres ثابت رغم كثرة النسخ.
- **العزل تحت الحِمل:** المهام الثقيلة خارج مسار الطلب (Streams) ⇒ لا تُشبع خيوط الـAPI.
- **التدهور الرشيق:** غياب مفتاح مزوّد ⇒ `409` واضح لا انهيار (لا Fallback بين المزوّدين، D‑16 — مقايضة مقبولة موثّقة).
- **الاسترداد:** DLQ + إعادة تشغيل العامل idempotent ⇒ لا ازدواج أثر.

## 6) ميزانية الموارد (تلميح أوّلي — يُعاير بالقياس)
| الخدمة | نقطة انطلاق |
|--------|-------------|
| App replica | 2 vCPU / 2 GB، عدّة عمّال Uvicorn/نسخة |
| Worker replica | 1–2 vCPU / 1–2 GB حسب نوع المهمة |
| PgBouncer | pool_mode=transaction، default_pool_size معاير على Postgres max_connections |
| Redis | ذاكرة كافية للـStreams + Cache مع سياسة إخلاء للـCache فقط |

## 7) نقاط توسعة القياس (Observability — مؤجّلة، جاهزة)
- `correlation_id` في كل طلب/حدث/خطأ.
- مظروف أحداث موحّد ⇒ إضافة Metrics/Tracing/Audit لاحقاً بلا إعادة هيكلة (قرار مؤجّل بطلبك).
- زمن دورة Outbox ومَلء DLQ مؤشّرا صحّة رئيسيان عند تفعيل المراقبة.
